<br>
I have created today's linux-next tree at<br>
git://git.kernel.org/pub/scm/linux/kernel/git/next/linux-next.git<br>
(patches at <a  rel="nofollow" href="http://www.kernel.org/pub/linux/kernel/next/">http://www.kernel.org/pub/linux/kernel/next/</a> ).  If you<br>
are tracking the linux-next tree using git, you should not use "git pull"<br>
to do so as that will try to merge the new linux-next release with the<br>
old one.  You should use "git fetch" and checkout or reset to the new<br>
master.<br>
<br>
You can see which trees have been included by looking in the Next/Trees<br>
file in the source.  There are also quilt-import.log and merge.log<br>
files in the Next directory.  Between each merge, the tree was built<br>
with a ppc64_defconfig for powerpc, an allmodconfig for x86_64, a<br>
multi_v7_defconfig for arm and a native build of tools/perf. After<br>
the final fixups (if any), I do an x86_64 modules_install followed by<br>
builds for x86_64 allnoconfig, powerpc allnoconfig (32 and 64 bit),<br>
ppc44x_defconfig, allyesconfig and pseries_le_defconfig and i386, sparc<br>
and sparc64 defconfig and htmldocs. And finally, a simple boot test<br>
of the powerpc pseries_le_defconfig kernel in qemu (with and without<br>
kvm enabled).<br>
<br>

<br>

fix patches pending for the current merge release).<br>
